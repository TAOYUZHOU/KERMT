## MSE 与 MAE：除形式外，结果上常有何差别？

两者都是 **对逐步预测 \(\hat R_f\) 与真值 \(R_f\) 的逐点误差** 做平均，但优化目标不同：

- **MSE（均方误差）**  
  - 对大误差惩罚 **按平方放大**，优化器更强烈地惩罚离群点、尾部分位。  
  - 最优预测在噪声为高斯时与 **条件期望** 相关；在异方差或重尾标签下，解往往更 **向整体均值收缩**（若与其它项叠加，parity 上可出现更明显的「压扁」）。  
  - 梯度与误差成正比，大误差处更新更大。

- **MAE（平均绝对误差）**  
  - 对所有误差 **线性** 惩罚，对离群点相对不敏感，解更接近 **条件中位数** 类行为。  
  - 训练更稳、对异常点不放大，但可能在尾部 **分辨率略钝**（取决于数据）。  

**实践**：同一架构下，MSE 常在 **RMSE** 上略占优，MAE 在 **MAE 指标** 上更直接对齐；若关心极端 Rf 或重尾，可对比二者 parity 斜率与分位偏差。

## 方案 C V2（对比实验草案）

**动机**：与 **C v1**（Beta NLL + 基于预测 \(\mu\) 的步间 **relu 单调正则**）对照，观察「只学 **不同溶剂比例下的逐步真值映射**」时的行为差异。

**C V2 设定（概念）**

- **保留**：双向上下文 MLP + 原始 6 维描述符 + 序列逐步监督（teacher-forced 前向/反向上下文与现 `KermtBidirectionalCv1` 一致）。  
- **主损失**：改为 **MSE 或 MAE** 在逐步 \(\mu\)（或迭代后的 `rf_pred`，训练阶段与 `train_bidirectional_c_v1.py` 实现一致）与 **真值 \(R_f\)** 之间，而 **不再使用 Beta NLL** 作为主要项。  
- **不使用残差式单调正则**：令 **`lambda_fwd = lambda_rev = 0`**，即 **不** 对相邻步差分加 relu 惩罚，避免与「纯拟合比例—Rf 曲线」的目标混淆。  
- **对比维度**：  
  - 校准（parity 斜率、尾部偏差）是否更接近/远离 MSE 强基线；  
  - 序列上真值非单调时，是否比 C v1 更少「人为拉平」。

**实现与配置**

- 训练脚本：`tlc/scripts/train_bidirectional_c_v1.py`，字段 `regression_loss: mse | mae | beta_nll`。  
- **200 epoch + MSE + 无单调项** 示例：`tlc/configs/tlc_bidirectional_c_v2_mse_ep200.yaml`（`save_dir`: `tlc/results/bidirectional_c_v2_mse_ep200`）。

```bash
cd KERMT && python tlc/scripts/train_bidirectional_c_v1.py \
  --config tlc/configs/tlc_bidirectional_c_v2_mse_ep200.yaml
```

- 评测/Parity 与方案 B 一致：`python tlc/scripts/eval_and_plot_bidirectional_c.py --model_dir <save_dir>`。

## 方案 C v3–v5：设计意图 vs 现实现（编号自 v3 起独立，勿与上文 C v1 / C V2 混淆）

下文 **C v3 / v4 / v5** 指 **规划中的架构演进**；**不是** 仓库里已有配置名 `tlc_bidirectional_c_v1.yaml` 等的版本号延续。上文 **C V2** 仍指 **现脚本可跑的 MSE+无单调项对照**。

### 现实现（`KermtBidirectionalCv1`，即上文「方案 C」默认）

- **训练**：`prev_rf` / `next_rf` 来自 **真值** `rf_true`（teacher forcing），作为 MLP 的 **标量输入维**。  
- **推理**：无真值时用 **初值 + `n_refine` 轮迭代** 逼近邻域条件；**不是** 严格的从左到右自回归链，也 **没有**「前向 AR 一整条 + 反向 AR 一整条再平均」的显式两条解码器。  
- **与下列「用户原意」不一致**，需靠 **v4+** 另起实现或新脚本。

### 用户原意（备忘）

- **类自回归** 的 **next-value prediction**：沿溶剂路径逐步预测各比例点上的 \(R_f\)；可对目标位置做 **mask**。  
- **前向** 与 **反向** 各跑 **一轮** 因果/单向解码，再对同一步的预测做 **融合（如取均值）**。  
- **不** 强行在输入里塞 **真值** `prev_rf` / `next_rf`；训练与测试的条件分布应尽量对齐（可用 scheduled sampling 等）。

### 数据共用（data_v2）

- 新实验统一建议用 **时间划分后的训练集** 构造 **`data_v2`**：  
  **`merged_data_molecule_split_v2_rf_stratified_heavy70_descriptor_aug_extreme/cleaned/train_data.csv`**（由 **`literature_temporal_split.py`** 生成；年份可选 **`reference_year_authoritative.csv`**）。  
- 实现时用 **独立 yaml / 脚本名**（如 `..._c_v3_...`）与旧 bidirectional 实验区分。

### C v3（极简向量回归）

- **一个头** 直接输出 **\(n\) 维（或路径上 \(T\) 维）\(R_f\) 向量**，对应 **\(n\) 个（或 \(T\) 个）固定离散溶剂比例点**；等价「单头多输出」，**无** 逐步循环、**无** 邻域标量条件。  
- **不考虑** 双向上下文：即 **不采用** 现 `KermtBidirectionalCv1` 的双支路 + `prev_rf`/`next_rf` 架构。  
- **不考虑** 步间正则：无 \(\lambda_{\mathrm{fwd}}/\lambda_{\mathrm{rev}}\) 等 relu 形状项。  
- **主损失**：以 **MAE 或 MSE** 在向量上与真值对齐即可（与 v1/v2 的 Beta/步间项解耦）。

### C v4（双向语境 + 双因果解码 + ensemble，无步间正则）

- **两个因果 / 单向解码器**：例如 **沿路径从左到右** 一条、**从右到左** 一条；每步只依赖已解码侧（或因果 mask 下的上下文），**不** 用真值邻域作输入。  
- **融合**：对同一比例点两路预测 **ensemble**（常用 **逐点均值**，亦可学习权重）。  
- **考虑双向语境**：两条方向上的条件信息进入各自解码过程（或通过共享 encoder + 两解码头等变体实现，实现时定稿）。  
- **不引入** 步间正则架构：**\(\lambda_{\mathrm{fwd}}=\lambda_{\mathrm{rev}}=0\)**（或根本不实现该项）。

#### C v4：规划原文 vs 当前实现（`c_v3_c_v4_model.py` · `KermtDualCausalCv4`）

上文「**不** 用真值邻域作输入」来自 **理想表述**；与仓库里 **已实现** 的 C v4 **并不完全一致**，需单独备忘：

| 维度 | `plan.md` 理想表述 | 当前代码 |
|------|-------------------|----------|
| 邻域条件 | 训练/测试尽量对齐；可用 **scheduled sampling**，**不** 强塞真值 `prev_rf` / `next_rf` | **训练**：`forward_train` 使用 **teacher forcing**，条件为 **`rf_true[t-1]`、`rf_true[t+1]`（detach）**；**推理**：`predict_sequence` 使用 **上一步/下一步的模型预测** + 首尾边界 |
| 步间 relu 正则 | **无** | **无**（与规划一致） |
| 双解码 + ensemble | 两向各一轮再融合 | **`0.5 * (rf_fwd + rf_bwd)`**，与规划一致 |
| 首尾无邻域 | 需约定占位初值 | 标量 **`boundary_rf`（默认 0.5）**；训练与推理应对齐（见代码注释） |

**结论**：现实现是 **「双因果 MLP + 训练期 teacher forcing + 推理期自回归」**；存在 **训练时见真值邻域、推理时见预测邻域** 的分布差异，与规划里「不塞真值邻域」的**字面目标**不符。若要对齐该条规划，需要改为 **仅用已解码预测** 作为 `prev`/`next`，或实现 **scheduled sampling**（及相应损失与稳定性处理）。

#### C v4：目标实现约定（与当前 `KermtDualCausalCv4` 区分）

以下为 **期望** 的解码形式（**尚未**在仓库中完全按此实现），便于与上表「当前代码」对照：

- **条件信息**：每一步应能利用 **全长历史** 上的 \(R_f\) 信息（**不是** 仅用 \(t-1\) 一维真值）；与「全部历史拼接进输入」等价的目标，是让步间依赖 **可学习、可随距离衰减或增强**。  
- **双向两支**：**前向**（时间递增）与 **反向**（时间递减）两条解码路径，**共享同一 trunk**（共享主体网络），仅在因果 mask / 读出上区分；两路对同一步的预测再 **ensemble**（如均值）。  
- **Mask**：对 **尚未预测** 的位置（或 padding、无效步）在损失与注意力处 **mask 掉**，避免把占位符当真值优化。

**表示层选型（定稿倾向）**

- **MLP 逐步扩维**（\(L +\) 全长历史 \(R_f\) 拼接）：实现上 **维数随步数线性增长**，网络宽深与归一化都要跟着调，**工程与优化负担大**。  
- **RNN**：用定长隐状态压历史，存在 **信息瓶颈**；长溶剂路径上早期比例对后期的影响可能被 **过度压缩**。  
- **Transformer（推荐）**：用 **位置编码 + 自注意力**（或 **Decoder** 在 **因果 mask** 下）显式建模步间依赖，**无** RNN 式单隐状态瓶颈，也 **无** MLP 式输入维爆炸；与下文 **方案 D2**（因果注意力双塔）同一技术族。C v4 目标实现可表述为：**GROVER 编码器得到分子/步特征后 → 共享 Transformer 解码骨干（两向不同因果 mask 或正反序各跑一遍）→ 逐步线性读出 \(R_f\)**。

### C v5（在 v4 上引入步间正则）

- 在 **C v4** 的架构上 **正常引入** 步间正则（与现 `scheme_c_loss` 类似：对相邻步预测差分的 **relu 单调** 或你选定的形状先验），\(\lambda\) **可大于 0**。  
- 用于约束沿溶剂路径的 \(R_f\) 行为；与 **v4** 的对比即「同结构下，加/不加步间先验」。

### 一览

| | 上下文与结构 | 步间正则 |
|---|----------------|----------|
| **C v3** | 单头 \(n\)（或 \(T\)）维 \(R_f\)，**无** 现双向邻域架构 | **无** |
| **C v4** | 双因果解码 + ensemble，**有** 双向语境设计 | **无** |
| **C v5** | 同 **v4** 路线 | **有** |

**多头注意力的「多头」** 与 **「输出层 \(T\) 维向量」** 不是同一概念。C v4 **目标实现** 已倾向 **Transformer 式序列建模范式**（见上节）；与 v3 的「单向量并行回归」正交。

## 方案 D（注意力序列建模范式 · 规划）

在 **C v3–v5** 的 MLP + 标量邻域（或并行向量）路线之外，单独规划 **基于序列注意力** 的变体，用于消融「显式 teacher-forcing 标量」与「隐空间融合多步信息」的差异。

### D1 — Encoder 式自注意力（全序列可见）

- 将路径上每步嵌入为 **`h_t`**（由 `x[t]` 或可学习位置编码得到），堆叠为 **`H ∈ R^{T×d}`**。  
- 使用 **Transformer Encoder**（或等价多层自注意力 + FFN），**无因果 mask**：每一步可 attend 到 **所有** \(t'\)。  
- 读出：每步 **Linear → \(R_f\)**，或与 **MLM 式 mask 训练** 结合（实现时定稿）。  
- **语义**：用 **软融合** 替代显式 `prev_rf` / `next_rf` 标量；训练与推理均为 **并行一步前向**（无逐步自回归依赖），与 **C v3** 的「并行预测」更接近，但归纳偏置不同。

### D2 — 因果注意力（mask 只看左侧 / 只看右侧）

- **前向塔**：标准 **因果 mask**（位置 \(t\) 只能看见 \(t' \le t\)），对应「从左到右」的信息流；  
- **反向塔**：在 **reverse 顺序的序列** 上施加因果 mask，或 **mask 只看 \(t' \ge t\)**，对应「从右到左」；  
- 两塔输出在逐步上 **ensemble**（如均值），与 **C v4** 的「双解码 + 融合」对齐，但 **邻域信息** 由 **attention 权重** 学习，而非显式一维标量输入。  
- **可选**：与 **C v5** 结合，在注意力读出后的 **\(\hat R_f\) 序列** 上仍加 **步间 relu 正则**。  
- **与 C v4 目标**：上文 **C v4 目标实现** 所选的 **Transformer 因果双塔 + 共享骨干** 与 **D2** 同构；实现时可把 **D2** 视为 C v4 的 **注意力版本** 落点，而非与 C v4 并列的另一条孤立方案。

### 与现代码关系

- **未实现**：当前仓库仅有 **C v3 / C v4**（`c_v3_c_v4_model.py`）与 **KermtBidirectionalCv1**；**方案 D** 需新模块与训练脚本。  
- **评测**：C v3/v4 可用 `eval_and_plot_bidirectional_c.py --plan c_v3|c_v4` 生成 parity（与 bidirectional_c v1 共用绘图管线）。
